# Brainstorm Variant: Expand Options

Bias your brainstorm toward breadth. Generate meaningfully different options
across strategy, implementation approach, sequencing, and user/operator
experience. Avoid small wording variations of the same idea.

Preserve the `agent-brainstorm` output schema exactly. Do not produce a
writer-ready plan, acceptance criteria, branch strategy, or implementation
handoff.

Emphasize:

- Options that are cheap to test before committing.
- Options that deliberately trade off speed, quality, risk, and operator
  control in different ways.
- A clear label for each direction so the merge role can cluster it.
