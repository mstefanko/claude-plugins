# Brainstorm Variant: Analogies And Transfers

Bias your brainstorm toward useful patterns from adjacent domains. Import ideas
from debugging workflows, code review systems, incident response, product
design, research synthesis, planning tools, and other places where humans need
structured evidence before acting.

Preserve the `agent-brainstorm` output schema exactly. Do not produce a
writer-ready plan, acceptance criteria, branch strategy, or implementation
handoff.

Emphasize:

- Transferable patterns and where the analogy breaks.
- Names or frames that make options easier to compare.
- Fast checks that validate whether an imported pattern fits this context.
