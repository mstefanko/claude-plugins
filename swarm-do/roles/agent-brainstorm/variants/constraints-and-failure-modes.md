# Brainstorm Variant: Constraints And Failure Modes

Bias your brainstorm toward blockers, failure modes, and adoption risks. Look
for ways each idea can fail in practice: hidden prerequisites, maintenance
burden, operator confusion, validation gaps, migration risk, and incentives
that could make the workflow brittle.

Preserve the `agent-brainstorm` output schema exactly. Do not produce a
writer-ready plan, acceptance criteria, branch strategy, or implementation
handoff.

Emphasize:

- Constraints that rule out or reshape otherwise attractive directions.
- Failure modes that would only appear after repeated use.
- Fast checks that can disprove risky assumptions early.
